## Image and Media Risks

| Area | Risk | Beta decision |
| --- | --- | --- |
| Vehicles | Large showcase media/stage may be visually heavy on phones. | Requires screenshot QA; not a current blocker from bundle output. |
| Character detail | Portrait/detail media plus dense sections may increase perceived load. | Can ship if screenshot QA passes. |
| Weapons/modules | Many cards use visual assets/placeholders. | Acceptable with current lazy route split. |
| Build Planner | Mostly UI/data density rather than images. | Can ship as prototype. |
| News thumbnails | External source/thumbnail patterns should be reviewed for source/privacy posture. | Keep legal/source caveat. |

## Current Assessment

Performance is acceptable for a static beta candidate. The main remaining risks are perceived mobile performance, image composition, and route density, not failed build budgets.

## Post-Beta Improvements

- Investigate main chunk splitting if the shared chunk grows beyond 150 kB gzip.
- Review character detail chunk dependencies and data imports.
- Add route-level loading skeletons only where measured as useful.
- Consider image dimension/format audit after asset ownership and hosting policy are settled.
- Add Lighthouse or WebPageTest checks after preview deployment exists.
